# VulnMain 产品手册

## 1. 文档说明

- 产品名称：`VulnMain 漏洞管理平台`
- 适用对象：安全工程师、研发工程师、项目负责人、系统管理员
- 文档目的：帮助用户快速掌握平台核心功能与标准操作流程

---

## 2. 产品简介

VulnMain 是一款企业级漏洞管理平台，覆盖漏洞从发现、分派、修复到复测关闭的全生命周期，并联动项目、资产、团队、用户和周报能力，实现安全治理闭环。

核心价值：

- 全流程闭环管理（漏洞创建 -> 修复 -> 复测 -> 关闭）
- 多角色协同（安全、研发、管理）
- 可追踪可审计（时间线、评论、关注者）
- 数据化运营（仪表盘与周报）

---

## 3. 角色与权限

| 角色 | 角色代码 | 主要能力 |
|---|---|---|
| 超级管理员 | `super_admin` | 全部权限 |
| 管理员 | `admin` | 全部权限（可维护系统、用户、项目、资产、漏洞） |
| 安全工程师 | `security_engineer` | 提交/分派漏洞、复测、资产管理 |
| 研发工程师 | `dev_engineer` | 处理分配给自己的漏洞，执行修复 |
| 普通用户 | `normal_user` | 最小可见权限（按授权查看） |

## 4. 登录与首页

### 4.1 登录方式

- 本地账号密码登录
- LDAP/AD 登录（需管理员先完成配置）
- Google OAuth 登录（需管理员先完成配置）

### 4.2 首页仪表盘

登录后可查看：

- 漏洞总体状态分布
- 最近漏洞动态
- 工程师排名
- 待处理/即将到期事项

不同角色看到的数据范围不同（全局 / 个人 / 分配视角）。

## 5. 核心业务流程（重点）

## 5.1 漏洞提报（安全工程师）

入口：`项目详情 -> 漏洞管理 -> 添加漏洞`

建议填写规范：

- 漏洞标题：问题 + 影响对象
- 漏洞地址：可复现 URL / 接口地址
- 漏洞描述：复现步骤、影响、边界（支持 Markdown）
- 修复建议：可执行的修复方案（支持 Markdown）
- 指派人：明确修复责任人
- 修复期限：可管理、可考核

## 5.2 漏洞修复（研发工程师）

常见状态流转：

`未修复/待处理 -> 修复中 -> 已修复`

研发应在漏洞详情中补充必要说明，便于复测和审计。

## 5.3 漏洞复测（安全工程师/管理员）

复测时支持提交验证材料，并保存为复测结果：

- 文字说明（Markdown）
- 图片上传（支持编辑器内上传）
- 支持 `Ctrl+V`（Windows/Linux）或 `Cmd+V`（macOS）直接粘贴截图

推荐复测记录模板：

1. 复测时间与环境
2. 验证步骤
3. 实际结果与预期对比
4. 结论（通过/不通过）
5. 证据截图

复测结论：

- 通过：漏洞状态进入关闭/完成
- 不通过：打回未修复，进入新一轮修复

## 5.4 协作机制

- 评论：可查看该漏洞的用户均可评论
- 关注者（Watcher）：可通过邮箱添加，便于跨团队同步
- 时间线：自动记录关键操作，满足追踪与审计需求

## 6. 模块操作指南

## 6.1 项目管理

功能：

- 项目创建、编辑、删除
- 项目成员维护
- 项目维度漏洞与资产统计

推荐实践：

- 按业务系统/产品线拆分项目
- 明确项目负责人
- 在项目创建时同步成员和周期

## 6.2 资产管理

功能：

- 资产新增/编辑/删除
- Excel 模板导入、批量导出
- 按环境、重要性、标签管理

资产信息建议最少包含：

- 资产名称、IP/域名、端口
- 资产类型、环境、重要性

## 6.3 漏洞管理

功能：

- 列表筛选（状态、严重程度、关键词）
- 漏洞详情、状态流转、时间线
- 评论与关注者管理

建议的状态治理：

- 设定修复 SLA（按严重程度分层）
- 定期清理长期未流转漏洞
- 使用复测材料提升关闭质量

## 6.4 团队管理

功能：

- 团队创建与成员维护
- 团队漏洞视图（支持多条件筛选）
- 基于团队/成员的漏洞可见性控制

## 6.5 用户管理

功能：

- 用户创建、编辑、禁用/启用
- 角色分配与密码重置

建议：

- 严格控制管理员数量
- 按职责分配最小权限

## 6.6 系统设置

可配置内容：

- 系统基础信息（系统名、公司名等）
- 邮件服务（通知与周报）
- LDAP/AD、Google OAuth
- 上传限制（大小、类型）
- 安全策略（密码策略等）

## 6.7 周报管理

功能：

- 周报数据聚合
- PDF 预览与下载
- 手动发送、定时发送

## 7. 标准操作建议（SOP）

## 7.1 日常使用（安全团队）

1. 每日查看仪表盘与到期漏洞
2. 新增漏洞按规范填写并分派
3. 跟进“修复中/已修复”漏洞
4. 复测时提交文字+截图证据
5. 关闭漏洞并沉淀评论/时间线

## 7.2 日常使用（研发团队）

1. 每日查看分配给我的漏洞
2. 明确修复计划并更新状态
3. 修复后标记已修复并补充说明
4. 对复测不通过项快速回归修复

## 8. 常见问题（FAQ）

## 8.1 复测如何上传截图？

在复测弹窗的 Markdown 编辑器中：

- 点击“图片”按钮上传，或
- 直接 `Ctrl+V/Cmd+V` 粘贴截图

## 8.2 为什么我看不到某些漏洞？

通常由权限与可见性控制导致，请检查：

- 当前角色权限
- 是否为项目成员/团队成员
- 是否被添加为该漏洞关注者

## 8.3 资产无法删除怎么办？

若资产下存在漏洞，系统会阻止删除。请先处理关联漏洞或做资产迁移。

## 9. 术语说明

- 漏洞生命周期：漏洞从发现到关闭的完整处理路径
- Watcher：漏洞关注者，可接收进展并参与协作
- SLA：服务等级目标，这里常指修复时限目标
